# 无限摄制社团官方网站开发计划

本计划旨在从零构建一个功能完善的社团官方网站，涵盖用户认证、双层级访问控制、内容管理及后台管理功能。

## 阶段一：项目初始化与基础设施搭建
### 1.1 项目结构搭建
- 创建前后端分离的项目结构（Monorepo 或 独立目录）。
- **前端**：初始化 Vue 3 + TypeScript + Vite 项目，安装 Element Plus、Pinia、Vue Router、Axios。
- **后端**：初始化 Node.js + Express + TypeScript 项目，配置 ESLint、Prettier。
- **数据库**：设计 MySQL 数据库 Schema（用户、角色、作品、任务、评论等表结构）。

### 1.2 基础服务配置
- **后端**：
    - 配置 Express 服务器，集成 Helmet（安全）、Cors（跨域）、Morgan（日志）。
    - 搭建数据库连接模块（使用 TypeORM 或 Sequelize 或 Prisma）。
    - 封装统一的响应格式和全局错误处理中间件。
- **前端**：
    - 配置全局样式（SCSS/Tailwind）、路由守卫。
    - 封装 Axios 请求拦截器（处理 Token）。

## 阶段二：用户认证与权限系统（核心）
### 2.1 后端开发
- **用户模型**：实现用户注册、登录、密码加密（Bcrypt）、JWT 签发与验证。
- **权限控制**：实现 RBAC（基于角色的访问控制）中间件，区分访客、社员、管理员权限。
- **接口**：注册、登录、获取用户信息、刷新 Token。

### 2.2 前端开发
- **页面**：开发登录页、注册页（包含表单验证）。
- **状态管理**：使用 Pinia 管理用户信息和登录状态。
- **交互**：实现登录跳转、权限路由拦截。

## 阶段三：外部公开页面与作品展示
### 3.1 页面开发
- **首页/社团介绍**：展示社团历史、架构、核心成员（响应式布局）。
- **作品展示**：实现瀑布流/网格布局的作品列表，支持分类筛选（摄影/摄像/后期）。
- **作品详情**：展示作品大图/视频、元数据（作者、时间、描述）。
- **SEO**：配置动态 Meta 标签。

### 3.2 后端支持
- **作品接口**：公开的作品列表与详情查询接口（支持分页、筛选）。

## 阶段四：内部社员系统
### 4.1 作品管理
- **上传功能**：前端集成文件上传组件（支持拖拽、预览），后端实现文件接收与存储（开发环境暂存本地，预留 OSS 接口）。
- **图片处理**：集成 Sharp 等库进行自动压缩、缩略图生成。

### 4.2 任务与交流
- **任务系统**：管理员发布任务，社员查看及提交成果。
- **交流区**：实现简单的帖子发布、评论与回复功能。
- **个人中心**：社员查看个人作品、任务记录及编辑资料。

## 阶段五：管理后台与运维
### 5.1 后台功能
- **用户管理**：社员审核列表、权限修改、封禁操作。
- **内容审核**：作品与评论的审核流。
- **数据统计**：集成 ECharts 展示用户增长、作品上传等图表。

### 5.2 测试与优化
- **测试**：编写关键业务的单元测试（Jest）。
- **安全**：配置 Rate Limiting（限流）、CSRF 防护。
- **部署准备**：编写 Dockerfile，配置 GitHub Actions CI/CD 流程（草案）。

## 关键技术栈确认
- **前端**：Vue 3, TypeScript, Vite, Element Plus
- **后端**：Node.js, Express, MySQL, Redis (可选)
- **存储**：本地文件系统 (开发阶段) / 阿里云 OSS (生产阶段)
